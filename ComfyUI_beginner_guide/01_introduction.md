# 第1章 はじめに

## このガイドのゴール

このガイドを最後まで読み終えたとき、あなたは次のことができるようになっています。

1. ComfyUI の画面を見て、どこに何があるか分かる
2. 自分の好きなプロンプト（言葉）から画像を生成できる
3. 自分の好きな雰囲気の音楽を生成できる
4. 「あれ？ 動かないな」となったときに、自分でそれを切り分けられる

## ComfyUI とは何か（30秒で分かる説明）

画像生成 AI のツールです。`Automatic1111 WebUI` や `Forge` などと違って、**「ノード」** と呼ばれる箱と、**「線」** だけで構成された画面が特徴です。

- **ノード** = 「1つの仕事をする箱」  
  例: モデルを読み込む箱／プロンプトを数値に変える箱／実際に画像を作る箱
- **線（ワイヤー）** = ノードからノードへデータを流すパイプ

レゴブロックを組み立てるように、自分の好きな「処理の流れ（ワークフロー）」を作れるのが ComfyUI の強みです。

> 💡 **最初の体験では、自分でブロックを組む必要はありません。** ComfyUI には「テンプレート」と呼ばれる、出来合いのワークフローが用意されています。このガイドでは基本そのテンプレートを使います。

## 前提条件

このガイドは、次の状態から始まることを想定しています。

- [x] ComfyUI が起動している
- [x] ブラウザで `http://localhost:8188/` を開くと画面が表示される

ComfyUI のインストール手順自体はこのガイドの範囲外です（環境によって全く異なるため）。すでに動いているという前提で進めます。

## このガイドの読み方

- **章は順番に読む前提です。** 後の章は前の章で説明した用語を使います。
- **画面のスクリーンショット** がたくさん出てきます。「赤枠で囲った部分」の指示が無ければ、画面全体の雰囲気をざっと見るだけで十分です。
- コードのような部分は **そのままコピペ可能** です。

## 用語の最低限のルール

このガイドでは、画面の表示と本文を一致させます。

| 本文中の表記 | 意味 |
|---|---|
| `「実行する」ボタン` | 画面に「実行する」と書かれているボタン（英語版だと "Queue Prompt"） |
| `Kサンプラー` ノード | ノードの上に「Kサンプラー」と書かれている箱 |
| `screenshots/01_xxx.png` | 添付されているスクリーンショット |

> 💡 **英語UIになっている方へ** — 設定で日本語UIに切り替えできます。手順は [第2章 画面ツアー → UIを日本語に切り替える](02_ui_tour.md#uiを日本語に切り替える) を参照してください。なお、英語UIのままでも各ノードの位置は同じなので、このガイドの手順はそのまま使えます。
