# 第9章 困ったときに

つまずきやすいポイントを症状から逆引きできるようにまとめました。

## 症状から探す

| 症状 | 移動先 |
|---|---|
| 「実行する」を押しても何も起きない | [→ ボタン無反応](#ボタン無反応) |
| 「モデルが見つかりません」が出る | [→ モデル不足ダイアログ](#モデル不足ダイアログ) |
| `out of memory` 系のエラーで止まる | [→ VRAM不足](#vram不足) |
| ノードが赤く光ってエラー | [→ ノード赤エラー](#ノード赤エラー) |
| 音は出来たが品質が悪い | [→ 音の品質改善](#音の品質改善) |
| 画像が変な構図・崩れている | [→ 画像が崩れる](#画像が崩れる) |
| ブラウザで画面が白い | [→ ComfyUI 自体が動いてない](#comfyui-自体が動いてない) |
| Models サイドバーにファイルが出ない | [→ ファイルが認識されない](#ファイルが認識されない) |

---

## ボタン無反応

押した瞬間、左サイドバーの **Queue** アイコンに数字バッジが付くはずです。付かない場合：

1. **画面右下にトースト（赤い通知）が出ていないか** 確認 — エラーメッセージが書いてある
2. **ComfyUI を起動したターミナル** を見る — Python のスタックトレースが流れていれば、それが原因
3. ノードの**ワイヤー（線）が外れていないか** 確認 — モデル/CLIP/VAE のいずれかが未接続だと黙って失敗することがある

> 💡 **接続が外れた場合** は、出力側の点（ソケット）から入力側の点へドラッグすればつなぎ直せます。

## モデル不足ダイアログ

```
モデルが見つかりません
グラフを読み込む際に、次のモデルが見つかりませんでした
```

**原因**: 必要なファイルが `models/` 配下に置かれていない、または ComfyUI に認識されていない。

**対処**:
1. ダイアログ内の **「ダウンロード」ボタン** を押す（推奨）
2. または **コマンドラインで配置** （第6章方法B）
3. 配置済みなのに出る場合は、**ブラウザを Ctrl+R でリロード**
4. それでもダメなら、**ファイル名のスペル** を確認。`stable-audio-open-1.0.safetensors` が `stable_audio_open_1.0.safetensors` になっていたりする

## VRAM不足

エラー例：
```
torch.cuda.OutOfMemoryError: CUDA out of memory.
HIP out of memory.
```

**原因**: GPUのVRAM が足りない。

**対処**（効果が大きい順）:
1. **バッチサイズを 1 にする**
2. **画像なら解像度を下げる**（1024 → 768 など）
3. **音声なら秒数を短くする**（47秒 → 10秒）
4. **他のアプリを閉じる**（ブラウザの動画タブなどが VRAM を食う）
5. **ComfyUI を `--lowvram` フラグ付きで起動** （Docker版なら起動コマンドに `--lowvram` を追加）
6. **モデルのアンロード**: ターミナルで `Ctrl+C` し再起動 — メモリリークが疑わしい時

> 💡 **VRAM の使用状況** は ComfyUI の左下にも `V: 14` のような形で表示されています。

## ノード赤エラー

ノードの周囲が **赤く光る** 場合、そのノードでエラーが発生しています。

**確認手順**:
1. 赤いノードを **左クリック** で選択
2. 画面下部または右下の通知エリアに **赤いトースト** が出る
3. ターミナルログで該当ノードのエラーを探す

**よくあるパターン**:

| エラーキーワード | 原因 |
|---|---|
| `KeyError: 'sd_xl_base_...'` | チェックポイントが見つからない |
| `Expected ... but got ...` | ワイヤーの型が合っていない（例: 画像出力をモデル入力に繋いでいる） |
| `RuntimeError: CUDA error` | VRAM 不足、または GPU ドライバ問題 |
| `Lengths of ... do not match` | バッチサイズの不整合（複数のノードでバッチ数を揃える） |

## 音の品質改善

「音は出るがイマイチ」のとき：

| 症状 | 対処 |
|---|---|
| ノイズだらけ | ステップ数を増やす（30→50） |
| プロンプトが効いてない | cfg を上げる（5→7） |
| 単調 | プロンプトに具体的な楽器名を入れる |
| ボーカルが混じる（Stable Audio） | ネガティブに `vocals, singing` を入れる |
| 楽器がはっきりしない（ACE-Step） | プロンプトのタグを増やす（5個→10個） |

> 💡 Stable Audio で **`euler`** など SDXL 系のサンプラーを選ぶと音が崩壊します。デフォルトの `dpmpp_3m_sde_gpu` を使いましょう。

## 画像が崩れる

| 症状 | 対処 |
|---|---|
| 顔・手が破綻する | 解像度を下げる（512×512 ではなく 1024×1024 を使う）／ネガティブに `bad anatomy, extra fingers` |
| ぼやけ続ける | ステップを増やす（20→30）／cfg を上げる |
| 同じ構図しか出ない | シードを `randomize` に戻す |
| 真っ黒・真っ白 | VAE の接続を確認、または別の VAE を試す |
| プロンプトが反映されない | ポジティブとネガティブを **逆に書いていないか** 確認 |

## ComfyUI 自体が動いてない

ブラウザで `http://localhost:8188/` を開いても **白い画面** または **接続エラー**：

1. **ComfyUI が起動しているか確認**:
   ```bash
   curl -s http://localhost:8188/system_stats
   ```
   JSON が返ってくれば起動中、`Connection refused` なら停止中。

2. **Docker 版の場合**:
   ```bash
   docker ps | grep comfyui
   ```
   コンテナが running になっていなければ：
   ```bash
   docker compose up -d
   ```

3. **ポートが他のアプリに使われていないか**:
   ```bash
   ss -tlnp | grep 8188
   ```

## ファイルが認識されない

`models/checkpoints/` にファイルを置いたのに ComfyUI の Models サイドバーに出てこない：

1. **ファイルの拡張子** が `.safetensors` `.ckpt` `.pt` のいずれかか確認
2. **ファイルサイズが 0 でないか**（途中で切れたダウンロード）
3. **ComfyUI のサーバー再起動** が必要な場合あり：
   - Docker 版: `docker compose restart`
   - 直接実行版: ターミナルで Ctrl+C → 再起動
4. **シンボリックリンク** で別ディスクから繋いでいる場合、ターゲットの実体があるか確認
5. **権限**: ファイルを root で作成していて ComfyUI ユーザーから読めない、など

## どうしてもダメなとき

1. **画面の右下にあるバージョン情報** や **左下のステータス** をスクリーンショット
2. **ComfyUI を起動したターミナルの最後の20行** をコピー
3. **どのテンプレート / プロンプトで** 起きたかメモ

これだけ揃えると、フォーラムや GitHub Issue で質問しやすくなります。
