# 第5章 プロンプトを変えて遊ぶ

第4章で動かしたデフォルトワークフローを **そのまま** 使い、プロンプトと設定を少しずつ変えながら自分の意図に近づけていきます。 **新しいモデルや新しいノードは追加しません。**

## 練習のキホン：1度に1か所だけ変える

良い絵が出たり出なかったりしたとき、**「何が効いたのか」** が分かるようにするのが大事です。

> 💡 **2か所同時に変えると、どちらが効いたか分からなくなります。** 1か所変える → 実行 → 結果を見る、を繰り返しましょう。

## 練習1：プロンプトの「重み付け」

単語や句に `()` で囲んで重みを付けられます。

```
a beautiful sunset, (cherry blossoms:1.4), mountain background, soft light
```

- `(cherry blossoms:1.4)` → 「桜」を **1.4倍** 強く意識させる
- `(low quality:1.5)` → ネガティブ側で「低品質要素を強く避ける」

### 重みの目安

| 値 | 効果 |
|---|---|
| 0.5 | かなり弱める |
| 0.8 | 少し弱める |
| 1.0 | デフォルト |
| 1.2 | 少し強調 |
| 1.5 | はっきり強調 |
| 1.8〜2.0 | 効きすぎて崩れることがある |

## 練習2：シードを固定して構図を比較する

第4章で `生成後の制御` を `randomize` にしていると、毎回違う構図が出ます。「この構図のまま色だけ変えたい」というときは：

1. `Kサンプラー` の **シード** の数字を覚える（例: 42）
2. **生成後の制御** を `fixed` に変更
3. プロンプトの一部だけ変えて再実行

→ 構図はほぼそのままで、変えた部分だけが反映されます。

## 練習3：解像度の縦横比を変える

`空の潜在画像` の幅・高さを変えると、画像のアスペクト比が変わります。SDXL で安定する組み合わせはこちら。

| サイズ | 用途 |
|---|---|
| 1024 × 1024 | スクエア（汎用） |
| 1152 × 896 | 横長・風景 |
| 896 × 1152 | 縦長・ポートレート |
| 1216 × 832 | やや横長 |
| 832 × 1216 | やや縦長 |

> 💡 **無理に 2048×2048 などにしないこと。** SDXL の学習解像度から外れて崩れやすく、VRAM も足りなくなります。大きい画像が欲しい時は **アップスケール** を別ステップで行います（このガイドの範囲外）。

## 練習4：複数枚一気に作る（バッチ）

「とにかくたくさん見て選びたい」とき、`空の潜在画像` ノードの **バッチサイズ** を 4 にしてみてください。

- バッチサイズ = 4 → 1回の実行で4枚生成
- 各画像はそれぞれ違うシード（randomize時）になる
- 生成時間と VRAM 消費が増える点に注意

VRAM が足りなくてエラーになる場合は、バッチサイズ 1 のまま **「キュー」** に複数回入れるという手もあります（`実行する` ボタンを連打すれば、その分だけ順番に実行されます）。

## プロンプトの定型句チートシート

迷ったら、ポジティブの末尾にこれらを足すと品質が上がりやすいです。

```
masterpiece, best quality, highly detailed, sharp focus, 8k
```

逆にネガティブには：

```
text, watermark, signature, low quality, blurry, lowres, jpeg artifacts, bad anatomy
```

## 「同じワークフローを後で使いたい」を保存する

良いセッティングが見つかったら保存しておきましょう。

1. 画面左上の **ロゴ → Workflow → Save**
2. ファイル名をつけて保存（`.json` ファイル）
3. 次回 **左サイドバーの Workflows → Load** で呼び出せる

> 💡 **生成された PNG ファイル自体にも、ワークフローが埋め込まれています。** その PNG をキャンバスに **ドラッグ & ドロップ** すると、ワークフローが復元されます。後で「あの設定どうだったっけ」となっても安心。
